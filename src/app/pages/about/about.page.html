About Gastowntest

What is Gastowntest?

Gastowntest is an Angular v21 Server-Side Rendering (SSR) application that demonstrates best practices for modern Angular development using the Smart/Presentational component pattern.

Smart/Presentational Pattern

This application follows a strict architectural pattern that separates concerns between two types of components:

Smart Components (Pages)

  • Located in src/app/pages/
  • Named with .page.ts suffix
  • Handle business logic and data fetching
  • Inject services via dependency injection
  • Manage state with Angular signals
  • Pass data DOWN to presentational components
  • Handle events UP from presentational components

Presentational Components (UI)

  • Located in src/app/components/
  • Named with .component.ts suffix
  • Pure UI rendering only
  • Receive data via input() signals
  • Emit events via output() signals
  • No service injection (except utilities)
  • No business logic
  • Highly reusable and testable

Angular v21 Features

This application leverages the latest Angular v21 features:

  • Standalone Components - No NgModules required
  • Signal APIs - Reactive state with signal(), computed(), effect()
  • Input/Output Signals - Modern input() and output() functions
  • Built-in Control Flow - @if, @else, @for in templates
  • Built-in SSR - Native server-side rendering support
  • Client Hydration - Seamless SSR to client transition
  • TransferState - Prevents duplicate API calls between server and client

Technology Stack

  • Angular v21
  • TypeScript 5.9
  • RxJS 7.8
  • Express.js 5.1 (SSR server)
  • SCSS for styling
  • Vitest for testing
